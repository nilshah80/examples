OAuth2 Token Flow with HMAC-SHA256 + AES-256-GCM in WebAssembly

üîê External Client with HMAC: This example uses HMAC-SHA256 signature for authentication (Step 2 only). A unique per-session storage key is derived at runtime using HKDF(WASM_STATIC_SECRET, session_id, client_id) to encrypt the session key before storing it in sessionStorage. JavaScript and browser dev tools can only see the encrypted session key, never the plaintext. Each session produces a different storage key. All cryptographic operations (ECDH, HKDF, AES-GCM, HMAC) run inside the WASM module.

üì¶ Storage Strategy

sessionStorage is used (better than localStorage) because:

  • Cleared when browser tab closes (more secure)
  • Not shared across tabs (isolation)
  • Perfect for temporary session data

The session key is encrypted with a per-session key derived via HKDF from the WASM-embedded secret and session_id, ensuring the browser never sees the plaintext key and each session has a unique storage encryption key.
